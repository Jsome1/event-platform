# Event Registration Platform 🚀

ეს არის სრულყოფილი ვებ-აპლიკაცია ღონისძიებების მართვისა და რეგისტრაციისთვის. პროექტი მოიცავს როგორც კლიენტის მხარეს (Frontend), ასევე სერვერულ ნაწილს (Backend) მონაცემთა ბაზასთან ერთად.

🔗 **პროექტის ნახვა (Live):** [Event Platform Live](https://jsome1.github.io/event-platform/)
💻 **რეპოზიტორი:** [GitHub Repository](https://github.com/Jsome1/event-platform.git)

## ✨ ფუნქციონალი

* **ღონისძიებების დათვალიერება:** მთავარ გვერდზე წარმოდგენილია აქტიური ივენთების სია.
* **ონლაინ რეგისტრაცია:** მომხმარებელს შეუძლია მარტივად დარეგისტრირდეს სასურველ ღონისძიებაზე.
* **მონაცემთა ვალიდაცია:** ფორმა ავტომატურად ამოწმებს შეყვანილი მონაცემების (Email, ტელეფონი) სისწორეს.
* **SQLite ბაზა:** ყველა რეგისტრაცია უსაფრთხოდ ინახება ლოკალურ მონაცემთა ბაზაში.
* **დინამიური UI:** ინტერფეისი რეაგირებს მომხმარებლის ქმედებებზე JavaScript-ის მეშვეობით.

## 🛠️ ტექნოლოგიები

* **Frontend:** HTML5, CSS3, JavaScript (ES6+)
* **Backend:** Node.js, Express.js
* **Database:** SQLite3
* **Version Control:** Git & GitHub

## 🚀 როგორ გავუშვათ პროექტი ლოკალურად

1.  დააკლონეთ რეპოზიტორი:
    ```bash
    git clone [https://github.com/Jsome1/event-platform.git](https://github.com/Jsome1/event-platform.git)
    ```
2.  გადადით პროექტის საქაღალდეში:
    ```bash
    cd event-platform
    ```
3.  დააინსტალირეთ საჭირო ბიბლიოთეკები:
    ```bash
    npm install
    ```
4.  გაუშვით სერვერი:
    ```bash
    node server.js
    ```
5.  გახსენით ბრაუზერში: `http://localhost:3000`

## 📂 პროექტის სტრუქტურა

* `server.js` - სერვერის მთავარი ფაილი და API ენდპოინტები.
* `db/` - მონაცემთა ბაზის ფაილები.
* `routes/` - სერვერული როუტების ლოგიკა.
* `public/` - კლიენტის მხარის ფაილები (ინტერფეისი).
* `.gitignore` - ფაილი ზედმეტი საქაღალდეების (მაგ. node_modules) გამოსარიცხად.
